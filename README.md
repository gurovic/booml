# booml
ML learning tasks

## Бэкенд

Серверная часть платформы **booml** построена на **Django 5** с использованием **Django REST Framework** для создания REST API и **Django Channels** для поддержки WebSocket-соединений. Аутентификация реализована через JWT-токены (библиотека `simplejwt`), а в качестве базы данных используется **PostgreSQL 16**. Такой стек позволяет надёжно хранить данные пользователей, курсов, контестов, задач и решений, обеспечивая при этом высокую производительность и масштабируемость.

Бизнес-логика вынесена в отдельный сервисный слой (`runner/services/`), что обеспечивает чистоту архитектуры и упрощает тестирование. REST-представления делегируют всю сложную работу сервисам: управлению правами доступа, оценке решений, управлению виртуальными машинами. Для асинхронной обработки задач (в первую очередь — проверки студенческих решений) используется **Celery** с брокером **Redis**, что позволяет не блокировать основной поток приложения при ресурсоёмких вычислениях.

Выполнение пользовательского кода происходит в изолированных **Docker-контейнерах** (runner-vm), внутри которых работают **Jupyter-ядра** с предустановленными ML-библиотеками (NumPy, Pandas, scikit-learn, Matplotlib и др.). Результаты выполнения доставляются обратно пользователю в реальном времени через WebSocket-канал на основе Django Channels и Redis channel layer. Такая схема гарантирует безопасность и воспроизводимость запуска студенческих ноутбуков в едином окружении.
